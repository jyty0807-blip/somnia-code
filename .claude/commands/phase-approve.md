Phase 리뷰 승인 후 진행도 업데이트 + 변경사항 기록을 자동 수행하는 스킬.

사용자가 리뷰 결과를 승인하면 이 스킬을 실행한다.

## 실행 절차

### 1단계: PROJECT-STATUS.md 업데이트
1. 해당 Phase 체크리스트 항목을 모두 `[x]`로 변경
2. Phase 제목에 ✅ 추가
3. 하단 Phase별 진행도 표에서 상태를 `✅ 완료`로 변경
4. 상단 "완료 항목" 섹션에 한 줄 요약 추가
5. 전체 진행도 퍼센트 + 프로그레스 바 업데이트 (Phase별 비중 합산)

### 2단계: CHANGELOG.md 업데이트
파일 최상단(# SOMNIA Changelog 바로 아래)에 새 Phase 항목을 추가한다:

```markdown
## Phase N: 제목 — YYYY-MM-DD

### 변경 파일
| 파일 | 변경 내용 |
|------|-----------|
| `파일경로` | 변경 설명 |

### 리뷰 수정사항
- 리뷰에서 지적받아 수정한 내용 목록
```

### 3단계: 완료 보고
```
Phase N 승인 완료.
- 진행도: XX% → YY%
- CHANGELOG.md 업데이트 완료
- 다음: Phase N+1 (제목)
```

### 4단계: 플랜 파일 동기화
현재 활성 플랜 파일(`.claude/plans/` 내)이 있으면, 완료된 Phase 정보가 플랜과 일치하는지 확인하고 필요시 업데이트한다 (제품 수, ID 목록 등).

## 중요 규칙
- 이 스킬은 사용자의 명시적 승인 후에만 실행한다.
- PROJECT-STATUS.md, CHANGELOG.md, 플랜 파일 — 이 3곳을 업데이트한다.
- 수정 후 다음 Phase를 자동으로 시작하지 않는다 — 보고만 하고 대기.
- **이 스킬을 거치지 않고 PROJECT-STATUS.md나 CHANGELOG.md를 수동으로 수정하는 것은 금지.**
